## ‚ú® Program Design Principles

Program design principles are guidelines that help developers create software that is **efficient, readable, scalable, and easy to maintain**.

### 1. Modularity
Modularity means dividing a program into **smaller, independent modules**, where each module performs a specific task.  
This makes the program easier to understand, test, debug, and reuse.

**Advantages:**
- Easier debugging
- Better code organization
- Parallel development possible

---

### 2. Abstraction
Abstraction focuses on **what a program does**, not **how it does it**.  
It hides complex implementation details and exposes only essential features to the user.

**Example:**  
Using a function without knowing its internal logic.

---

### 3. Encapsulation
Encapsulation is the practice of **binding data and methods together** in a single unit, usually a class.  
It also restricts direct access to data to prevent accidental modification.

**Benefits:**
- Data security
- Better control over program flow
- Reduced complexity

---

### 4. Reusability
Reusability allows existing code to be used again in different programs or projects, reducing development time and errors.

**Techniques for reusability:**
- Functions
- Classes
- Libraries
- Modules

---

### 5. Readability and Maintainability
Readable code uses:
- Meaningful variable names  
- Proper indentation  
- Comments  

Maintainable programs are easier to update, debug, and extend in the future.

---

### 6. Efficiency
Efficiency refers to optimal usage of:
- **Time** (fast execution)
- **Memory** (minimal resource usage)

Efficient programs are especially important for large-scale applications.

## üóÑÔ∏è Database Management with Python

Database management with Python allows efficient **storage, retrieval, and manipulation of data** for applications and ML systems.

---

### 1. Database
A database is an **organized collection of structured data** stored electronically.

**Types of Databases:**
- Relational databases (SQL)
- Non-relational databases (NoSQL)

---

### 2. Relational Databases
Relational databases store data in **tables (rows and columns)**.

**Examples:**
- MySQL
- PostgreSQL
- SQLite

---

### 3. Python Database Libraries
Python provides several libraries to interact with databases:
- `sqlite3` ‚Äì lightweight, built-in database
- `psycopg2` ‚Äì PostgreSQL connector
- `MySQLdb` ‚Äì MySQL connector
- `SQLAlchemy` ‚Äì ORM-based database management

---

### 4. CRUD Operations
CRUD represents the four basic database operations:
- **Create** ‚Äì Insert data
- **Read** ‚Äì Retrieve data
- **Update** ‚Äì Modify data
- **Delete** ‚Äì Remove data

These operations are performed using SQL queries through Python.

---

### 5. Transactions
Transactions ensure **data consistency and reliability**.

Key concepts:
- Commit ‚Äì saves changes permanently
- Rollback ‚Äì restores data in case of failure

---

### 6. ORM (Object Relational Mapping)
ORM allows developers to interact with databases using **Python objects instead of SQL queries**.

**Advantages:**
- Cleaner code
- Reduced SQL errors
- Faster development

---

### 7. Applications of Database Management with Python
- Web applications
- Data analytics
- Machine learning pipelines
- Business information systems
